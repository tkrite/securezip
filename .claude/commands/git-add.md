lead-developer が以下の手順でファイルをステージングします：

1. `git status` で現在の変更状況を確認
2. 変更内容を分析し、論理的なまとまりを把握
3. ステージング対象の決定：
   - $ARGUMENTS が指定されている場合: そのファイル/パターンのみ追加
   - 指定がない場合: 全変更内容を確認し、適切な単位で `git add` を実行
4. `git add [対象]` を実行
5. `git status` でステージング結果を確認
6. ステージング結果を報告：
   - ステージングしたファイル一覧
   - 残りの未ステージングファイル（あれば）
