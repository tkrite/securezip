lead-developer が以下の手順でマージを準備・実行します：

1. `git branch` で現在のブランチを確認
2. `git status` で未コミットの変更がないか確認
   - 未コミット変更がある場合はコミットまたはstashを促す
3. `git log --oneline [対象ブランチ]..HEAD` でコミット履歴を確認
4. マージ情報を分析：
   - 現在のブランチ（マージ元）
   - マージ先（$ARGUMENTS で指定、デフォルトはmain）
   - コミット数
5. `git diff [対象ブランチ]..HEAD --stat` で差分サマリーを作成
6. **ユーザーに確認を求める**：
   ```
   【マージ確認】
   - マージ元ブランチ: [現在のブランチ]
   - マージ先ブランチ: [対象ブランチ]
   - 変更コミット数: [N]件
   - 変更ファイル数: [M]ファイル
   
   主な変更内容:
   [変更サマリー]
   
   マージを実行しますか？ (yes/no)
   ```
7. ユーザー承認後：
   - `git checkout [対象ブランチ]`
   - `git merge [元ブランチ] --no-ff -m "merge([scope]): [元ブランチ] into [対象ブランチ]"`
8. コンフリクト発生時：
   - `git diff --name-only --diff-filter=U` でコンフリクトファイルを特定
   - コンフリクト箇所を報告
   - 解決方法を提案
   - ユーザーの指示を待つ
9. マージ成功時：
   - マージ結果を報告
   - 元ブランチの削除要否を確認

**注意**: mainブランチへのマージは必ずユーザー承認を得る。承認なしにマージを実行しない。
