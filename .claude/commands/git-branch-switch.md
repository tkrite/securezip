lead-developer が以下の手順でブランチを切り替えます：

1. `git status` で未コミットの変更を確認
2. 未コミット変更がある場合：
   - 変更内容を報告
   - 選択肢を提示：
     - `stash`: 変更を一時退避して切り替え
     - `commit`: 変更をコミットしてから切り替え
     - `discard`: 変更を破棄して切り替え（要確認）
   - ユーザーの指示を待つ
3. `git branch -a` でブランチ一覧を表示
4. 切り替え先の決定：
   - $ARGUMENTS で指定されたブランチがあれば使用
   - なければブランチ一覧から選択を求める
5. `git checkout [ブランチ名]` を実行
6. 切り替え結果を報告：
   - 切り替え先ブランチ名
   - 最新コミット情報
   - 現在の状態
