lead-developer が以下の手順で新規ブランチを作成します：

1. `git branch` で現在のブランチ状況を確認
2. `git status` で未コミットの変更がないか確認
   - 未コミット変更がある場合は警告し、対処方法を提示
3. $ARGUMENTS から作成目的を分析し、ブランチ種別を判定：
   - 機能開発 → `feature/[機能名]`
   - バグ修正 → `fix/[issue番号]-[説明]` または `fix/[説明]`
   - 実験的開発 → `experimental/[実験名]`
4. CLAUDE.mdのブランチ命名規則に従い、ブランチ名を提案
5. **ユーザーに確認を求める**：
   ```
   【ブランチ作成確認】
   - 提案ブランチ名: [ブランチ名]
   - 作成元ブランチ: [現在のブランチ/main]
   - Worktree作成: [要/不要]
   
   この内容でブランチを作成しますか？
   変更がある場合はお知らせください。
   ```
6. ユーザー承認後、以下を実行：
   - 通常: `git checkout -b [ブランチ名]`
   - Worktree: `git worktree add ../project-[type]-[name] -b [ブランチ名]`
7. 作成結果を報告

**注意**: ユーザーの明示的な承認なしにブランチを作成しない。
