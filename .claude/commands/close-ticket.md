# チケットクローズコマンド

`./Documents/tickets/` 内の `$ARGUMENTS` で指定されたチケットをクローズし、対応する Git コミットを作成します。

## 前提条件

以下が完了していることを確認：
- [ ] 開発完了レビューが「合格」となっている
- [ ] ユーザーによる最終確認が完了している

## 実行フロー

### 1. クローズ可否の確認

チケットファイルを読み込み、クローズ可能か確認：

```bash
cat ./Documents/tickets/$ARGUMENTS
```

確認項目：
- ステータスが「In Progress」または「Review」であること
- 開発完了レビューが「合格」であること
- 全ての完了条件がチェック済みであること

### 2. ユーザー承認の確認

```
⚠️ チケットクローズ確認

チケット: [チケットID]
タイトル: [タイトル]

このチケットをクローズし、Gitコミットを作成します。
よろしいですか？ (yes/no)
```

**ユーザーの承認が得られた場合のみ続行します。**

### 3. チケットファイルの更新

#### 3.1 ステータス更新

チケット情報テーブルのステータスを更新：

```markdown
| **ステータス** | Closed |
```

#### 3.2 進捗履歴の追記

```markdown
| YYYY-MM-DD HH:MM | Closed | チケットクローズ - ユーザー承認済み | Claude Code |
```

#### 3.3 最終更新日時の更新

```markdown
*最終更新: YYYY-MM-DD HH:MM JST*
```

### 4. 開発ログの作成（オプション）

規模の大きい開発の場合、document-writer を呼び出して開発ログを作成：

```
開発ログの作成が推奨されます。作成しますか？ (yes/no)
```

### 5. Git コミットの実行

#### 5.1 変更のステージング

```bash
# 変更ファイルの確認
git status

# 関連ファイルのステージング
git add [変更ファイル群]
git add ./Documents/tickets/$ARGUMENTS
```

#### 5.2 コミットメッセージの作成

CLAUDE.md のコミットメッセージ規約に従ってメッセージを作成：

**開発チケット (ticket-dev-XXX) の場合:**
```
feat([scope]): [チケットタイトルの要約]

- [主な変更点1]
- [主な変更点2]
- [主な変更点3]

Closes #[チケット番号/Issue番号]
Ticket: ticket-dev-XXX
```

**修正チケット (ticket-fix-XXX) の場合:**
```
fix([scope]): [チケットタイトルの要約]

- [修正内容1]
- [修正内容2]

Root cause: [原因の簡潔な説明]

Fixes #[チケット番号/Issue番号]
Ticket: ticket-fix-XXX
```

#### 5.3 コミット実行

```bash
git commit -m "[コミットメッセージ]"
```

### 6. 完了報告

```
═══════════════════════════════════════════════════
✅ チケットクローズ完了
═══════════════════════════════════════════════════

チケット: [チケットID]
タイトル: [タイトル]
クローズ日時: YYYY-MM-DD HH:MM JST

───────────────────────────────────────────────────
Git コミット情報
───────────────────────────────────────────────────
コミットハッシュ: [ハッシュ値]
コミットメッセージ: [メッセージ概要]

───────────────────────────────────────────────────
変更ファイル
───────────────────────────────────────────────────
[変更ファイル一覧]

═══════════════════════════════════════════════════

💡 次のステップ:
- git push でリモートに反映
- 関連するIssue/PRがあれば更新
```

## エラーハンドリング

### クローズ不可の場合

```
❌ チケットをクローズできません

理由: [理由]
- 開発完了レビューが未実施です
- 未完了の完了条件があります

対応:
1. /review-ticket-completion [チケットファイル名] を実行してください
2. 全ての完了条件を満たしてから再度実行してください
```

### Git エラーの場合

```
⚠️ Git コミットに失敗しました

エラー内容:
[エラーメッセージ]

対応:
- 未ステージングのファイルがないか確認してください
- コンフリクトが発生していないか確認してください
```

## 注意事項

- ユーザーの承認なしにチケットをクローズしない
- コミット前に必ず変更内容を確認する
- 開発完了レビューが「差し戻し」の場合はクローズ不可
- コミットメッセージは規約に厳密に従う

## ロールバックが必要な場合

万が一、クローズ後に問題が発見された場合：

1. チケットのステータスを「Open」に戻す
2. 進捗履歴に再オープンの理由を記載
3. 必要に応じて `git revert` を実行
