lead-developer が以下の手順でリモートとの同期状態を確認します：

1. `git fetch --all` でリモート情報を取得
2. `git branch -vv` でローカルブランチとリモートの対応を確認
3. `git status` でローカルとリモートの差分を確認
4. 状態を分析・報告：
   ```
   【同期状態レポート】
   - 現在のブランチ: [ブランチ名]
   - トラッキング先: [origin/ブランチ名]
   - 状態: [同期済み / ローカルが先行 / リモートが先行 / 分岐]
   ```
5. 状態に応じた詳細情報：
   - ローカルが進んでいる場合: push可能なコミット数と内容
   - リモートが進んでいる場合: pull推奨、変更内容のサマリー
   - 分岐している場合: rebase/merge の選択肢を提示
6. 推奨アクションを提案：
   ```
   【推奨アクション】
   - [具体的なアクション提案]
   
   ※ push/pull の実行はユーザー判断となります
   ```

**注意**: このコマンドはpush/pullを自動実行しない（確認と提案のみ）。
