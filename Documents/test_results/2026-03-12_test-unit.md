## 検出された知見

### validateTarPaths の設計確認
テスト実行を通じて `validateTarPaths` の実際の動作を確認した。

**設計の確認事項:**
- `validateTarPaths` は「`..`」「`.`」「空コンポーネント」を除去（sanitize）して ZIP Slip を無力化する設計
- `../escape.txt` → sanitize → `escape.txt` → destination 配下 → エラーなし（意図通り）
- `/etc/passwd` → sanitize → `etc/passwd` → destination 配下 → エラーなし（意図通り）
- `decompressionFailed` がスローされるのは、sanitize 後のパスが destination 外を指す特殊ケースのみ

**App Sandbox 制約の影響:**
- テストターゲットは App Sandbox 下で動作するため `/usr/bin/tar` および外部プロセスを呼び出せない
- `decompressTar` は `validateTarPaths` 後に `runProcess(/usr/bin/tar)` を呼ぶが、App Sandbox で失敗する
- `..` を含む TAR に対しては `/usr/bin/tar` 自体が "Path contains '..'" エラーを返すため `compressionFailed` になる
- この制約はテスト環境固有であり、実際のアプリ動作（App Sandbox 外の通常 macOS アプリとして）には影響しない

### セキュリティ観点での評価
- ZIP Slip 対策は sanitize 方式で実装されており、`..` を含むパスでも destination 外へのファイル書き込みは防止されている（`testDecompress_pathTraversal_doesNotWriteFilesOutsideDestination` で確認）
- 絶対パスも sanitize によって相対パスに変換されるため、system ファイルへの上書きリスクはない
