# SecureZip for Mac - プロジェクト構造と開発計画

## プロジェクト概要

### 基本情報
- **プロジェクト名**: SecureZip for Mac
- **コードネーム**: SZ-2026
- **バージョン**: 1.0.0
- **Bundle ID**: com.tkrite.SecureZip
- **作成日**: 2026-02-27
- **更新日**: 2026-03-04
- **ステータス**: 開発中（Phase 2 進行中）

### 目的とビジョン
**目的**:
コマンドラインの知識がなくても、ビジネスレベルのセキュアなファイル送付ができるmacOSアプリケーションを提供する。GUIのみで暗号化圧縮・パスワード管理・Gmail送信までを一気通貫に完結させ、PPAPワークフローの安全性と利便性を両立する。

**ビジョン**:
非エンジニアのMacビジネスユーザーが、3ステップ以内で暗号化ファイルを安全に送信できる状態を実現する。パスワード自動生成・別送・履歴自動削除により、セキュリティインシデントのリスクを根本から低減する。

**対象ユーザー**:
macOS 13 Ventura以降を使用する非エンジニアのビジネスユーザー。Gmailアカウントを保有し、機密ファイルをメールで送付するケースが多いユーザーを主なターゲットとする。

## システムアーキテクチャ

### 技術スタック
#### アプリケーション
- **プラットフォーム**: macOS 13 Ventura 以降
- **開発言語**: Swift 5.9+
- **UIフレームワーク**: SwiftUI（macOS 13+）
- **アーキテクチャ**: MVVM + Service Layer
- **Bundle ID**: com.tkrite.SecureZip

#### 圧縮・暗号化
- **圧縮エンジン（ZIP）**: libarchive C API（`zip:encryption=aes256`オプション使用）
  - ZIP: libarchive C API経由（AES-256暗号化対応）
  - TAR系: macOS標準CLI（`tar`, `unzip`）をProcess経由で使用（継続）
  - 対応形式: ZIP / TAR.GZ / TAR.BZ2 / TAR.ZST
- **暗号化ライブラリ**: CryptoKit（macOS標準搭載）
  - パスワード生成: SecRandomCopyBytes
  - 公開鍵暗号: Curve25519（Phase 4実装予定）

#### データ永続化
- **ローカルDB**: Core Data（macOS標準搭載）
  - 送付履歴（SendHistoryEntity）- recipientEmailを非正規化して直接保存
  - アプリ設定（AppSettings）
  - **注意**: `.xcdatamodeld` 不在時はプログラム定義のNSManagedObjectModel + NSInMemoryStoreTypeでフォールバック
- **シークレット管理**: Keychain Services（macOS標準搭載）
  - OAuthトークン（Access Token / Refresh Token）
  - 暗号化パスワード（履歴IDと紐付け）

#### 外部連携
- **メール送信**: Gmail API v1（REST API / HTTPS）- 実装済み（2026-03-04）
- **OAuth認証**: Google Sign-In SDK + GTMAppAuth（SPM経由）- 実装済み（2026-03-04）
- **認証方式**: OAuth 2.0 + PKCE、スコープ: `gmail.send`のみ

#### 開発環境
- **IDE**: Xcode 15.0+
- **パッケージ管理**: Swift Package Manager（SPM）のみ
- **テストフレームワーク**: XCTest + XCUITest
- **ビルド**: Universal Binary（Apple Silicon / Intel）

## データベース設計

### エンティティ設計（実装済み）

#### SendHistoryEntity（非正規化設計）
recipientEmailを直接保持し、Recipientエンティティへの参照を持たない設計を採用（JOIN不要でシンプルに）。

| カラム名 | 型 | 制約 | 説明 |
|---------|-----|------|------|
| id | UUID | PK | 一意識別子 |
| recipientEmail | String | NOT NULL | 送付先メールアドレス（非正規化） |
| fileName | String | NOT NULL | 圧縮ファイル名 |
| originalFileNames | String | NOT NULL | 元ファイル名（JSON配列） |
| fileSize | Int64 | NOT NULL, DEFAULT 0 | ファイルサイズ（bytes） |
| format | String | NOT NULL, DEFAULT "zip" | 圧縮形式 |
| isEncrypted | Bool | NOT NULL, DEFAULT false | 暗号化有無 |
| sentAt | Date | NULL | 送信日時 |
| expiresAt | Date | NULL | 自動削除予定日時 |
| status | String | NOT NULL, DEFAULT "created" | ステータス（created/sending/sent/cancelled/failed） |
| createdAt | Date | NOT NULL | 作成日時 |

**設計判断**: v1.0で計画していたRecipientテーブルとの正規化設計から変更。HistoryServiceでrecipientEmailを直接保存する非正規化設計を採用した。理由: 送付先の管理が不要なPhase 1時点ではシンプルな設計が望ましく、Phase 2以降で必要に応じてRecipientエンティティを導入可能。

## 外部API仕様

### Gmail API（実装済み）
| メソッド | エンドポイント | 概要 | 認証 |
|---------|--------------|------|------|
| POST | `https://gmail.googleapis.com/gmail/v1/users/me/messages/send` | メール送信 | OAuth 2.0 Bearer Token |
| POST | `https://oauth2.googleapis.com/token` | トークンリフレッシュ | Client ID / Secret |

- **認証スコープ**: `https://www.googleapis.com/auth/gmail.send`（送信のみ、読み取り不可）
- **添付上限**: 25MB（これを超える場合は分割圧縮推奨）
- **リクエスト形式**: Base64URLエンコードされたMIMEメッセージ
- **現在の状態**: GmailAPIClient・GmailServiceともに本実装済み（2026-03-04）。MIME構築・401リトライ対応。KeychainからKeychainKey.gmailAccessTokenでトークン取得。

### OAuth 2.0認証フロー（実装済み）
1. アプリがGIDSignIn経由でOAuth認証画面を開く（Safari経由）
2. ユーザー認証後、reversed-client-IDスキームでコールバック受信（AppDelegate処理）
3. 認証コードをアクセストークン・リフレッシュトークンに交換
4. トークンをKeychain Servicesに保存
5. 以降はGTMAppAuthが自動でトークンリフレッシュ
6. SettingsViewModelがconnectedEmailを起動時にKeychain/GIDSignInから復元

### 既知の技術的問題
- **libarchive dylib バージョン不整合**: Homebrew libarchive 3.8.1がmacOS 15.0向けビルドのため、macOS 13/14環境でdylibロード失敗リスクあり。対応方針: デプロイターゲットを15.0に変更、またはlibarchiveをmacOS 13向けに再ビルドして静的リンク。
  - ヘッダー: `/opt/homebrew/opt/libarchive/include`（HEADER_SEARCH_PATHS設定済み）
  - ライブラリ: `/opt/homebrew/opt/libarchive/lib`（LIBRARY_SEARCH_PATHS設定済み）
- **Info.plist GIDClientID / URLスキームがプレースホルダー**: Google Cloud ConsoleでのOAuthクライアントID取得後に置き換えが必要
- **GoogleService-Info.plist未配置**: ダウンロード・配置・.gitignore追加が必要
- **実機でのOAuth認証フロー確認未実施**: GmailAPIClient・GmailServiceの本実装は完了しているが、実機動作確認が未実施
- **Core Dataフォールバック**: `.xcdatamodeld`不在時はNSInMemoryStoreTypeで動作

### 技術的制約事項
- libarchiveは7Z書き込み非対応（7Z形式は対象外）
- TAR系形式はAES-256暗号化非対応（暗号化はZIP形式のみ）
- Gmail API添付ファイル上限: 25MB
- App Sandboxにより、ユーザー選択ファイル以外へのアクセス不可
- Gmail送信のみ対応（Outlook / Yahoo Mail等は対象外）
